# Development Guide

## Environment

1. OS: Windows.
2. Compiler toolchain: MSVC (Visual Studio 2017 x64).
3. Qt: 5.12.12 (`D:/soft/Qt5.12.12/5.12.12/msvc2017_64`).
4. Build system: CMake.
5. IDE: VS Code with CMake Tools and C/C++ extension.

## Build Presets

Configure preset:

1. `default` (inherits `msvc-qt5-release`).

Build presets:

1. `default-debug-build`.
2. `default-release-build`.

## VS Code One-Click Build

1. Open workspace root.
2. Run `CMake: Configure` once if the project is newly opened.
3. Use Build command from CMake Tools.

## VS Code Startup Modes (F5)

Two launch profiles are available and should be used as standard startup methods:

1. `Debug CMake Target (MSVC)`
   - Trigger: press `F5` after selecting this profile.
   - Behavior: runs prelaunch task `CMake Build Debug`.
   - Program path: `${command:cmake.launchTargetPath}`.

2. `Release CMake Target (MSVC)`
   - Trigger: press `F5` after selecting this profile.
   - Behavior: runs prelaunch task `CMake Build Release`.
   - Program path: `${workspaceFolder}/build/cmake-msvc-release/Release/visualFrameInfo.exe`.

## Command-Line Build

From workspace root:

1. Configure:
   `cmake --preset msvc-qt5-release`
2. Build Debug:
   `cmake --build --preset default-debug-build`
3. Build Release:
   `cmake --build --preset default-release-build`

## Release Deploy (windeployqt)

Use this workflow to prepare a runnable folder under `install/Release`.

1. Build Release first:
   `cmake --build --preset default-release-build`
2. Run deploy script from workspace root:
   `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/deploy_release.ps1`
3. Verify deploy output:
   - `install/Release/visualFrameInfo.exe`
   - Qt runtime files and plugin folders generated by windeployqt
   - `install/Release/ffmpeg/ffmpeg.exe`
   - `install/Release/ffmpeg/ffprobe.exe`

## VS Code One-Click Release Deploy

1. Run task `Build And Deploy Release`.
2. The task chain runs in sequence:
   - `CMake Build Release`
   - `Deploy Release Package`

## Windows Installer Packaging (Inno Setup)

Use this workflow to package `install/Release` into a Windows installer executable.

Prerequisites:

1. Install Inno Setup 6.
2. Ensure `ISCC.exe` is available by one of the following ways:
   - Add Inno Setup installation path to `PATH`.
   - Set environment variable `INNO_SETUP_HOME` to Inno Setup install directory.
   - Use default install location (`C:/Program Files (x86)/Inno Setup 6` or `C:/Program Files/Inno Setup 6`).
   - Let packaging script auto-detect Inno Setup install location from Windows uninstall registry entries.

Command-line packaging:

1. Build and deploy release first:
   `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/deploy_release.ps1`
2. Build installer:
   `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/package_inno.ps1`
3. Output installer path:
   - `install/installer/visualFrameInfo_setup_<project-version>.exe`
   - `<project-version>` is auto-read from `CMakeLists.txt` `project(... VERSION ...)`

VS Code one-click packaging:

1. Run task `Build Deploy And Package Installer`.
2. The task chain runs in sequence:
   - `Build And Deploy Release`
   - `Package Inno Setup Installer`

## Installer Upgrade Validation Checklist

Use this checklist when validating overwrite upgrade installation behavior.

### A. Baseline install

1. Install previous version package (for example `visualFrameInfo_setup_0.1.0.exe`).
2. Confirm app starts from start menu shortcut and optional desktop shortcut.
3. Record install location and version shown in Programs and Features.

### B. Overwrite upgrade install

1. Run newer installer package directly (for example `visualFrameInfo_setup_0.1.1.exe`).
2. Confirm installer detects existing app and completes without manual cleanup.
3. Launch app and verify:
   - main window opens normally
   - ffmpeg analysis workflow still works
   - no Qt runtime missing DLL errors

### C. Shortcut validation

1. Start menu shortcut exists and points to latest install path.
2. Desktop shortcut behavior matches selected install option.
3. Launch from both shortcuts works without UAC or missing-file errors.

### D. Uninstall residual check

1. Uninstall from Programs and Features.
2. Confirm install directory is removed or only expected user-generated files remain.
3. Confirm start menu and desktop shortcuts are removed.
4. Confirm app entry is removed from Programs and Features.

## Runtime Dependency

1. The application searches bundled tools under `ffmpeg/ffprobe.exe` and `ffmpeg/ffmpeg.exe` (with upward directory fallback).
2. Keep `ffmpeg` directory alongside workspace/application deployment structure.

## Frame Preview Feature Notes

1. Frame preview is optional and controlled by `Frame Preview` toggle in timeline toolbar.
2. Toggle default is OFF and remains disabled unless a valid source video file exists.
3. Preview renders still images extracted by `ffmpeg` and does not provide embedded playback.
4. Preview cache files are generated under `logs/preview_cache` to reduce repeated extraction cost.
5. Rapid frame navigation uses async request handling and applies only the latest preview result.

## Benchmark Outputs

1. In-app `Run Benchmark` appends comparison rows to `logs/benchmark/benchmark_comparison.tsv`.
2. Per-run backend logs are written to `logs/benchmark/*_ffprobe.log` and `logs/benchmark/*_showinfo.log`.
3. Regular analysis runs append lightweight metrics to `logs/benchmark_metrics.tsv`.

## Common Troubleshooting

### A. Configure preset is not active

Symptoms:

1. Build command reports `No configure preset is active`.
2. CMake Tools cannot configure/build right after opening workspace.

Check order:

1. Confirm workspace settings in `.vscode/settings.json` include preset-related entries (`cmake.configurePreset`, `cmake.buildPreset`, `cmake.useCMakePresets`).
2. Run `CMake: Configure` once from command palette.
3. Verify status bar shows an active configure preset (`default` or inherited preset).
4. Retry Build/F5.

### B. Qt path mismatch or Qt not found

Symptoms:

1. Configure step fails with `Could not find a package configuration file provided by Qt5`.
2. Build fails because Qt5 targets (`Qt5::Core`, `Qt5::Widgets`) are unavailable.

Check order:

1. Open `CMakePresets.json` and confirm `CMAKE_PREFIX_PATH` points to local Qt install root.
2. Verify the path exists on disk, especially `.../msvc2017_64/lib/cmake/Qt5`.
3. Re-run configure command:
   `cmake --preset msvc-qt5-release`
4. If path changed, update preset and reconfigure before building.

### C. F5 launch failure (Debug/Release)

Symptoms:

1. F5 reports executable not found.
2. F5 starts but immediately exits due to missing build output.

Check order:

1. In launch profile selector, confirm target profile is intended one:
   - `Debug CMake Target (MSVC)`
   - `Release CMake Target (MSVC)`
2. Execute the matching task once manually:
   - `CMake Build Debug`
   - `CMake Build Release`
3. Confirm output exists:
   - Debug: `build/cmake-msvc-release/Debug/visualFrameInfo.exe`
   - Release: `build/cmake-msvc-release/Release/visualFrameInfo.exe`
4. If using Debug profile, ensure CMake launch target in status bar is `visualFrameInfo`.
5. Retry F5.

### D. Safe recovery sequence

Use this if state is inconsistent:

1. Close running debug session.
2. Run `CMake: Delete Cache and Reconfigure`.
3. Run `CMake Build Release` or `CMake Build Debug`.
4. Start with matching F5 launch profile.

### E. windeployqt was not found

Symptoms:

1. Deploy script fails with `Unable to locate windeployqt.exe`.

Check order:

1. Confirm `CMakePresets.json` contains valid `CMAKE_PREFIX_PATH`.
2. Confirm `<CMAKE_PREFIX_PATH>/bin/windeployqt.exe` exists.
3. If preset path is unavailable, add Qt bin to `PATH` and retry deploy task.

### F. Deploy output misses ffmpeg binaries

Symptoms:

1. `install/Release/ffmpeg` is missing.
2. Runtime analysis fails due to missing ffmpeg tools.
3. Frame preview toggle is enabled but no preview image can be extracted.

Check order:

1. Confirm workspace root contains `ffmpeg/ffmpeg.exe` and `ffmpeg/ffprobe.exe`.
2. Re-run `Deploy Release Package` task.

### G. Inno Setup compiler is not found

Symptoms:

1. Packaging script fails with `Unable to locate ISCC.exe`.

Check order:

1. Confirm Inno Setup 6 is installed.
2. Confirm `ISCC.exe` exists in installation directory.
3. Set `INNO_SETUP_HOME` or add ISCC directory to `PATH`.
4. Re-run `Package Inno Setup Installer` task.
